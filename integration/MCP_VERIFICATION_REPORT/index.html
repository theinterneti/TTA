    üîß Configuration Requirements¶

    For Docker MCP Toolkit Integration:¶

    1. Neo4j MCP Server Configuration: 

      docker run --rm --network host \
  -e NEO4J_URI="bolt://localhost:7687" \
  -e NEO4J_USERNAME="neo4j" \
  -e NEO4J_PASSWORD="neo4j_password" \
  mcp/neo4j-memory

    2. PostgreSQL MCP Server Configuration: 

      docker run --rm --network host \
  -e PGHOST="localhost" \
  -e PGPORT="5432" \
  -e PGDATABASE="tta_db" \
  -e PGUSER="tta_user" \
  -e PGPASSWORD="tta_password" \
  mcp/postgres

    3. Grafana MCP Server Configuration: 

      docker run --rm --network host \
  -e GRAFANA_URL="http://localhost:3000" \
  -e GRAFANA_API_KEY="<service_account_token>" \
  mcp/grafana

    For VS Code Integration:¶

    Create .vscode/settings.json: 

    {
  "mcp": {
    "servers": {
      "neo4j": {
        "type": "docker",

          "NEO4J_PASSWORD": "neo4j_password"
        }
      },
      "postgresql": {
        "type": "docker",

        "env": {
          "PGHOST": "localhost",
          "PGDATABASE": "tta_db",

          "PGPASSWORD": "tta_password"
        }
      },
      "grafana": {
        "type": "docker",

          "GRAFANA_URL": "http://localhost:3000",

  }

    ‚ö†Ô∏è Issues Identified & Troubleshooting¶

    Issue 1: MCP Server Tools Not Installed¶

    Problem: Command-line MCP tools are not globally installed Solution: 

    # Install PostgreSQL MCP Server
npm install -g @executeautomation/database-server

# Install Python uv for Neo4j MCP
pip3 install uv

# Install Grafana MCP Server (requires Go)
go install github.com/grafana/mcp-grafana/cmd/mcp-grafana@latest

    Issue 2: Grafana API Key Required¶

    Problem: Grafana MCP server requires service account API key Solution: 1. Access Grafana at http://localhost:3000 (admin/admin) 2. Go to Administration ‚Üí Service Accounts 3. Create service account with appropriate permissions 4. Generate API key and update configuration

    Issue 3: Redis MCP Server Selection¶

    Problem: Multiple Redis MCP implementations available Recommendation: Use direct Redis connection via environment variables: 

    REDIS_URL=redis://localhost:6379
